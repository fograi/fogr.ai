**Rewrite `src/routes/(public)/ad/[slug]/+page.server.ts`**

The current implementation passes `params.slug` directly to `/api/ads/${params.slug}` which does a UUID lookup. Replace this with direct Supabase queries and three-case redirect logic.

Import the slug utilities:

```typescript
import { redirect, error } from '@sveltejs/kit';
import { isUuidParam, parseSlugShortId } from '$lib/server/slugs';
```

The new load function handles four cases in order:

**Case 1: UUID parameter**

- Check `isUuidParam(params.slug)`.
- If true, query: `locals.supabase.from('ads').select('slug').eq('id', params.slug).maybeSingle()`.
- If no result, `throw error(404, 'Ad not found')`.
- If found, `throw redirect(301, '/ad/' + data.slug)`.
- This works for all ad statuses (active, pending, rejected, expired) per user decision -- do NOT filter by status on UUID redirect lookup.

**Case 2: Parse short ID from slug**

- Call `parseSlugShortId(params.slug)`.
- If null (invalid format), `throw error(404, 'Ad not found')`.

**Case 3: Lookup by short_id**

- Query Supabase directly (not via the API endpoint):
  ```typescript
  const { data: ad, error: dbError } = await locals.supabase
  	.from('ads')
  	.select(
  		'id, user_id, slug, title, description, category, category_profile_data, location_profile_data, price, currency, image_keys, status, created_at, updated_at, expires_at, firm_price, min_offer, auto_decline_message'
  	)
  	.eq('short_id', shortId)
  	.maybeSingle();
  ```
- If no result, `throw error(404, 'Ad not found')`.
- Check access: if ad is not active or is expired, only the owner can see it. Get the user via `locals.supabase.auth.getUser()`. If non-owner tries to view non-active/expired ad, `throw error(404, 'Ad not found')`.

**Case 4: Canonical slug check**

- If `params.slug !== ad.slug`, `throw redirect(301, '/ad/' + ad.slug)`.

**Case 5: Render the page**

- Keep the existing logic for building the `AdCard` mapped object, fetching owner conversations, and fetching moderation data.
- For the moderation query, use `ad.id` (not `params.slug`).
- Return the same shape as currently: `{ ad: mapped, moderation, isOwner, ownerMessages, offerRules }`.
- Add `slug: ad.slug` to the mapped AdCard object.

**Important implementation notes:**

- The auth check for non-public ads must mirror the existing behavior in `/api/ads/[id]` GET handler: non-active status and expired ads are only visible to the ad owner.
- Do NOT call `fetch('/api/ads/...')` anymore -- the page load queries Supabase directly. This avoids the need to modify the API endpoint's UUID-based cache keys.
- Keep the `locals.supabase.auth.getUser()` call for ownership check and moderation data, but only call it when needed (not for public active ads).

**Update `src/routes/api/ads/[id]/+server.ts` GET handler**

Add `slug` and `short_id` to the select query (line 184) so that any consumers of this API also get the slug:

```
.select('id, user_id, slug, short_id, title, description, ...')
```

This change is minimal -- just add the columns to the existing select string. The API endpoint continues to accept UUID-based `[id]` parameter for backward compatibility (used by internal API calls, cron worker, etc.).
