### Integration Defects

1. **County param mismatch in "View All" URL (Minor, NEW)**
   - **File:** `src/cron-worker.ts` line 407
   - **Bug:** `searchParams.set('county', search.county)` — uses `county` param
   - **Expected:** `searchParams.set('county_id', search.county)` — homepage reads `county_id` (line 24 of `src/routes/+page.server.ts`)
   - **Impact:** Saved search digest "View All" CTA lands on homepage with county filter silently dropped. Users see unfiltered results instead of county-specific matches.
   - **Affects:** EMAL-05, ENGR-04
   - **Fix:** Single-line change: `'county'` → `'county_id'`

### Integration Notes (Info)

1. **isNewAccount fail-open (Info)**: If env vars missing at deploy, `isNewAccount()` returns `false` (fail-open), so new-account ads get indexed. Documented as intentional design in `new-account.ts`. Affects: SEO noindex only, not the pending hold.

2. **Pre-migration slug backfill**: Ads predating Phase 1 migration without slugs cause 301 redirect chains in email links. `scripts/backfill-slugs.ts` exists but execution should be confirmed before launch. Affects: EMAL-02, EMAL-03 email link quality.
