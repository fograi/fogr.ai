**Migration 1: email_preferences (20260312_000018)**
Create `email_preferences` table per RESEARCH.md schema:

- id uuid PK default gen_random_uuid()
- user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
- email_type text NOT NULL (values: 'messages', 'search_alerts', 'ad_approved')
- suppressed boolean NOT NULL DEFAULT false
- created_at timestamptz NOT NULL DEFAULT now()
- updated_at timestamptz NOT NULL DEFAULT now()
- UNIQUE INDEX on (user_id, email_type)
- RLS enabled. Policy: authenticated users SELECT/UPDATE own rows only (auth.uid() = user_id). Service role bypasses RLS automatically.

**Migration 2: saved_searches (20260312_000019)**
Create `saved_searches` table per RESEARCH.md schema:

- id uuid PK default gen_random_uuid()
- user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
- name text (nullable, optional user-given name)
- category text (nullable = any category)
- county text (nullable = any county)
- locality text (nullable = any locality)
- query text (nullable = no keyword filter)
- notify boolean NOT NULL DEFAULT true
- last_notified_at timestamptz NOT NULL DEFAULT now()
- created_at timestamptz NOT NULL DEFAULT now()
- INDEX on user_id
- Partial INDEX on notify WHERE notify = true
- Composite INDEX on (category, county) for efficient matching
- RLS enabled. Policy: authenticated users ALL on own rows (auth.uid() = user_id) WITH CHECK.

**send.ts -- Resend REST API wrapper**
Create a fire-and-forget `sendEmail()` function per RESEARCH.md Pattern 1:

- Accepts: `env` object (needs RESEND_API_KEY), `params: { to, subject, html, text?, headers? }`
- Calls `POST https://api.resend.com/emails` with Bearer token auth
- From address: `fogr.ai <eolas@fogr.ai>` (per user decision -- note: use ASCII "fogr.ai" not fada characters in the from field for email deliverability)
- Returns `string | null` (Resend email ID or null on failure)
- On failure: `console.error('email_send_failed', { status, body })` and return null -- never throw
- On network error: `console.error('email_send_error', { error, to })` and return null
- Export the `SendEmailParams` type

The `env` parameter type should be: `{ RESEND_API_KEY: string; PUBLIC_SUPABASE_URL: string; SUPABASE_SERVICE_ROLE_KEY: string; UNSUBSCRIBE_SECRET: string }` -- export this as `EmailEnv` type. This avoids coupling to the full cron worker Env type.

**unsubscribe.ts -- HMAC token utilities**
Create unsubscribe token functions per RESEARCH.md Pattern 2:

- `generateUnsubscribeToken(secret, userId, emailType)` -> Promise<string>
  - Uses Web Crypto `crypto.subtle.sign` with HMAC-SHA256
  - Token format: `{base64url_signature}.{base64_payload}` where payload is `userId:emailType`
- `verifyUnsubscribeToken(secret, token)` -> Promise<{ userId, emailType } | null>
  - Uses Web Crypto `crypto.subtle.verify` for timing-safe comparison
  - Returns null on invalid/tampered tokens
- `buildUnsubscribeHeaders(unsubscribeUrl)` -> Record<string, string>
  - Returns `{ 'List-Unsubscribe': '<url>', 'List-Unsubscribe-Post': 'List-Unsubscribe=One-Click' }`
  - Per RFC 8058 and RESEARCH.md Pattern 3
- Helper functions: `importKey`, `bufferToBase64url`, `base64urlToBuffer` (internal, not exported)

**preferences.ts -- Email preference check**
Create preference check per RESEARCH.md Pattern 5:

- `isEmailSuppressed(env: EmailEnv, userId: string, emailType: string)` -> Promise<boolean>
  - Queries email_preferences table via Supabase REST API
  - Filters: user_id eq, email_type eq, suppressed eq true
  - Returns true if row exists (suppressed), false otherwise
  - Fail-open: if query errors, return false (send the email if unsure)
  - Uses the same `supabaseHeaders` pattern as cron-worker.ts
- `suppressEmail(env: EmailEnv, userId: string, emailType: string)` -> Promise<boolean>
  - Upserts email_preferences row with suppressed=true
  - Uses Supabase REST with `Prefer: resolution=merge-duplicates` for upsert on (user_id, email_type)
  - Returns true on success, false on error
- `unsuppressEmail(env: EmailEnv, userId: string, emailType: string)` -> Promise<boolean>
  - Updates existing row to suppressed=false, or does nothing if no row exists
  - Returns true on success
