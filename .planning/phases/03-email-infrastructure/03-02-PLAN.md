Build the complete unsubscribe flow: a machine-facing API endpoint for RFC 8058 one-click unsubscribe (used by Gmail/Yahoo/Outlook), and a browser-facing page for users who click the unsubscribe link in an email.

Purpose: EMAL-06 requires every email to include a working unsubscribe link. RFC 8058 compliance is required by Gmail (Jun 2024), Yahoo (Jun 2024), and Outlook (May 2025) for deliverability. Without this, emails may go to spam or lack the "Unsubscribe" button in email headers.
Output: 1 API route (POST-only), 1 SvelteKit page route (load + form actions + Svelte UI).
</objective>

  <action>
Create `src/routes/api/unsubscribe/+server.ts` with a POST handler only.

This endpoint is called by Gmail/Yahoo/Outlook servers when a user clicks the one-click unsubscribe button in their email client. Per RFC 8058 and RESEARCH.md Pitfall 2, this MUST:

- Accept POST requests only
- Return HTTP 200 with empty body (no redirects, no HTML)
- Process the unsubscribe immediately

Implementation:

1. Extract `token` and `type` from URL search params (the List-Unsubscribe header URL includes these)
2. Get UNSUBSCRIBE_SECRET from `platform.env.UNSUBSCRIBE_SECRET`
3. Call `verifyUnsubscribeToken(secret, token)` from `$lib/server/email/unsubscribe`
4. If token is invalid: return 400 with `{ error: 'Invalid token' }`
5. If token is valid: extract userId and emailType
6. Validate that emailType is one of the suppressible types: 'messages', 'search_alerts', 'ad_approved'
7. Call `suppressEmail(env, userId, emailType)` from `$lib/server/email/preferences`
8. Return 200 with empty JSON `{}` regardless of suppressEmail success (fail-open for the user's benefit)
9. Log: `console.log('unsubscribe_api_processed', { userId, emailType })`

Build the env object for suppressEmail from platform.env:

```typescript
const env = {
	RESEND_API_KEY: platform?.env?.RESEND_API_KEY ?? '',
	PUBLIC_SUPABASE_URL: platform?.env?.PUBLIC_SUPABASE_URL ?? '',
	SUPABASE_SERVICE_ROLE_KEY: platform?.env?.SUPABASE_SERVICE_ROLE_KEY ?? '',
	UNSUBSCRIBE_SECRET: platform?.env?.UNSUBSCRIBE_SECRET ?? ''
};
```

Note: No CSRF check needed -- this endpoint is called by external email servers, not the app's own frontend.
Note: No auth check needed -- the HMAC token IS the authentication (it proves the user received the email).

**+page.server.ts -- Server load + form actions**

This page is loaded when a user clicks the unsubscribe link in an email from their browser.

`load` function:

1. Extract `token` and `type` from `url.searchParams`
2. If either is missing: return `{ error: 'Missing parameters', success: false, emailType: null }`
3. Get UNSUBSCRIBE_SECRET from `platform.env.UNSUBSCRIBE_SECRET`
4. Call `verifyUnsubscribeToken(secret, token)` from `$lib/server/email/unsubscribe`
5. If invalid: return `{ error: 'Invalid or expired link', success: false, emailType: null }`
6. Build env object from platform.env (same pattern as Task 1)
7. Call `suppressEmail(env, userId, emailType)`
8. Return `{ success: true, emailType: result.emailType, error: null }`

Define a `resubscribe` form action:

1. Extract token and type from the form data (pass them as hidden inputs in the form)
2. Verify token (same as load)
3. Call `unsuppressEmail(env, userId, emailType)` from `$lib/server/email/preferences`
4. Return `{ resubscribed: true, emailType }`

Map email type slugs to display names:

- 'messages' -> 'message notifications'
- 'search_alerts' -> 'saved search alerts'
- 'ad_approved' -> 'listing approval notifications'

**+page.svelte -- Confirmation UI**

Per user decision: "Confirmation page after unsubscribe: 'You have been unsubscribed from [type] emails' with re-subscribe link"

Simple, minimal page:

- If `data.error`: show error message with a "Back to fogr.ai" link
- If `data.success`: show confirmation:
  - Heading: "Unsubscribed"
  - Body: "You have been unsubscribed from {emailType display name}."
  - "You will no longer receive these emails."
  - Re-subscribe form: a `<form method="POST" action="?/resubscribe">` with hidden token/type fields and a text link "Changed your mind? Re-subscribe"
- If form result shows resubscribed: "You have been re-subscribed to {emailType display name}."
- Footer link: "Return to fogr.ai" -> "/"

Style: Use existing layout from (public) group. Keep minimal -- this is a utility page, not a marketing page. Inline Tailwind classes matching the existing site aesthetic.

Set page metadata: `<svelte:head><title>Unsubscribe - fogr.ai</title></svelte:head>`
