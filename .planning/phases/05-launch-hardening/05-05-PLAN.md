Create `e2e/mobile-audit.test.ts` with Playwright tests at 375px x 812px viewport (iPhone SE / iPhone 12 Mini dimensions).

**Test setup:**

```typescript
import { test, expect } from '@playwright/test';

test.use({ viewport: { width: 375, height: 812 } });
```

**Test cases:**

1. **Homepage browse at 375px:**
   - Navigate to `/`
   - Assert no horizontal scrollbar: `await expect(page.locator('body')).toHaveCSS('overflow-x', 'visible')` or check `document.documentElement.scrollWidth <= 375`
   - Assert hero text is visible and not truncated
   - Assert ad cards are single-column (not overlapping)
   - Assert filter controls are accessible
   - Assert the anti-dealer tagline is visible if results exist
   - Take screenshot: `await page.screenshot({ path: 'test-results/mobile-homepage.png' })`

2. **Ad view page at 375px:**
   - Navigate to a known ad page (use a mock or seed listing slug)
   - Assert title, price, description visible
   - Assert safety accordion is present (collapsed)
   - Assert no horizontal overflow
   - Assert action buttons (Share, Save, Report) don't overflow
   - Take screenshot

3. **Post form at 375px:**
   - Navigate to `/post` (will redirect to login if not authenticated -- test with auth if possible, otherwise check the login redirect works)
   - If authenticated: assert form fields are full-width, not cut off
   - Assert step indicators fit in viewport
   - Assert private-seller note is visible
   - Take screenshot

4. **Safety page at 375px:**
   - Navigate to `/safety`
   - Assert all sections render without overflow
   - Assert text is readable (font size >= 14px equivalent)
   - Take screenshot

5. **About page at 375px:**
   - Navigate to `/about`
   - Assert card grid collapses to single column
   - Assert no horizontal overflow
   - Take screenshot

6. **Footer at 375px:**
   - Navigate to `/`
   - Scroll to footer
   - Assert footer links are accessible and not overlapping
   - Assert Safety tips link is visible

**After running tests, fix any blocking issues found.** Common 375px issues to check and fix:

- **Horizontal overflow:** Add `overflow-x: hidden` to body/main if content bleeds. Check for elements with fixed pixel widths > 375px. Fix with `max-width: 100%` or responsive units.
- **Text truncation:** Check that titles/descriptions use `word-wrap: break-word` or `overflow-wrap: break-word`.
- **Button overflow:** Action buttons in ad view rail may need wrapping at 375px. Add `flex-wrap: wrap` if needed.
- **Filter controls:** Browse page filters may need stacking at 375px. Check existing @media queries handle this.
- **New elements from plans 02/03:** The safety accordion, browse tagline, private-seller note, and footer trust line should all work at 375px.

Apply CSS fixes to `src/app.css` for global issues, or to specific component `<style>` blocks for component-specific issues. Use existing breakpoint convention: `@media (max-width: 640px)` for mobile. Only fix blocking issues -- no style redesign per CONTEXT.md.

Test existing breakpoint at 640px is sufficient for most issues. For 375px-specific fixes, add `@media (max-width: 400px)` if the 640px breakpoint doesn't cover narrow-phone edge cases.
