```typescript
export const BIKE_TYPES_TO_SEED = [
	'road',
```

Define county-specific content. Create a `COUNTY_FLAVOUR` record mapping each of the 32 county IDs (from ireland_counties.json) to:

```typescript
export type CountyFlavour = {
	demonym: string; // e.g., "Dub", "Corkonian", "Kerryman"
	landmark: string; // e.g., "Phoenix Park", "the English Market", "Sliabh Luachra"
	terrain: string; // e.g., "canal paths", "harbour roads", "mountain trails"
```

  <name>Task 2: Create the seed script with batch insertion and error handling</name>

Create `scripts/seed-listings.ts` -- a standalone Node.js script runnable via `npx tsx scripts/seed-listings.ts`.

**Environment:**

- Reads `PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from environment (or `.env` file)
- Also reads an optional `SEED_USER_ID` env var for the system account UUID. If not provided, the script should create or look up a system user.

**Imports:**

```typescript
import { createClient } from '@supabase/supabase-js';
import { generateAdSlug } from '../src/lib/server/slugs';
import { buildLocationProfileData } from '../src/lib/location-hierarchy';
import { generateAllListings } from './seed-data';
```

Note: Since this runs via `npx tsx`, the TypeScript path aliases (`$lib`) won't work. Use relative imports from the scripts directory.

**Flow:**

1. Load environment variables. If using `.env`, use `dotenv` (already in devDependencies) or read manually. Check that Supabase URL and service key are present. Exit with error if missing.

2. Create Supabase client with service role key:

```typescript
const supabase = createClient(url, key, {
	auth: { persistSession: false, autoRefreshToken: false }
});
```

3. Look up or create the system account. Query `auth.users` via admin API for a user with email `eolas@fogr.ai`. If not found, create one via `supabase.auth.admin.createUser({ email: 'eolas@fogr.ai', email_confirm: true })`. Store the user ID.

4. Generate all listings via `generateAllListings()`.

5. For each listing, construct the full insertion object:

```typescript
{
  user_id: systemUserId,
  title: listing.title,
  description: listing.description,
  category: 'Bikes',
  price: listing.price,
  currency: listing.currency,
  status: 'active',
  image_keys: [],
  firm_price: false,
  email: 'eolas@fogr.ai',
  slug: generateAdSlug(listing.title, listing.countyName, 'Bikes'),
  short_id: slug.slice(-8), // extracted from generated slug
  category_profile_data: {
    version: 1,
    profile: 'bikes',
    subtype: listing.subtype,
    bikeType: listing.bikeType,
    condition: listing.condition,
    sizePreset: listing.sizePreset
  },
  location_profile_data: buildLocationProfileData([listing.countyId]),
  expires_at: new Date(Date.now() + 14 * 24 * 60 * 60 * 1000).toISOString()
}
```

6. Generate all slugs upfront. Deduplicate any collisions (if two listings generate the same slug, regenerate with a suffix). This avoids database constraint violations.

7. Batch insert in chunks of 50 (Supabase REST has payload limits). Use `supabase.from('ads').insert(chunk)` for each batch. If a batch fails with error code `23505` (unique constraint), fall back to individual inserts with retry (max 3 attempts per listing, regenerating slug each time).

8. Log progress: `Inserted batch X/Y (Z listings)` for each batch. At the end, log the total: `Seeded N listings across 32 counties.`

9. Handle errors gracefully. If a listing fails after 3 retries, log the failure and continue with the next listing. Do not abort the entire script for one failure.

**Script should be idempotent-safe:** Before inserting, check if seed listings already exist by querying `ads` where `user_id = systemUserId` and `status = 'active'`. If count > 100, log a warning and exit: `"Seed listings already present (N found). Delete existing seeds before re-running."` This prevents accidental double-seeding.
