**Step 2: Create `src/lib/server/slugs.ts`**

This module exports three functions:

1. `generateAdSlug(title: string, countyName: string | null, categoryName: string | null): string`
   - Use `slugify(title, { lower: true, strict: true, trim: true })` for text-to-slug conversion. This handles fadas/diacritics (a->a, e->e, o->o, u->u, i->i) and strips emoji/non-alphanumeric per user decision.
   - Remove stop words from the slugified title. Use this curated list (per research recommendation):
     ```
     Articles: a, an, the
     Conjunctions: and, or, but, nor, so, yet
     Prepositions: in, on, at, to, for, of, with, by, from, up, about, into, over, after
     Pronouns: i, me, my, we, us, our, you, your, he, him, his, she, her, it, its, they, them, their
     Be verbs: is, am, are, was, were, be, been, being
     Common verbs: has, have, had, do, does, did, will, would, could, should, may, might
     Other: this, that, these, those, not, no, if, as
     ```
   - If title produces empty slug after stop word removal (all non-Latin chars / emoji-only title), fall back to `slugify(categoryName)` or `'listing'` if no category.
   - Truncate at ~60 characters on a word boundary (find last `-` before 60 chars).
   - Slugify the countyName separately. If null/empty, omit it from the slug.
   - Generate 8-char short ID using `customAlphabet('abcdefghijklmnopqrstuvwxyz0123456789', 8)` from nanoid.
   - Assemble: `[titleSlug, countySlug, shortId].filter(Boolean).join('-')`

2. `parseSlugShortId(slug: string): string | null`
   - Extract last 8 characters of the slug.
   - Validate they match `/^[a-z0-9]{8}$/`. Return null if invalid.

3. `isUuidParam(param: string): boolean`
   - Test against `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i`.

Export the `STOP_WORDS` set and `MAX_TITLE_SLUG_LENGTH` constant for testing.

Also export `extractShortId(slug: string): string` (extracts last 8 chars without validation -- used internally and by backfill).

Follow codebase conventions: named exports, camelCase functions, `string | null` return pattern for parsing, SCREAMING_SNAKE_CASE for constants. File should have `'use strict'` implicit (ESM). Use `import type` where applicable.

**Step 3: Create `src/lib/server/slugs.spec.ts`**

Write Vitest unit tests covering:
- Basic slug generation: `"Trek Domane Road Bike"` with county `"Dublin"` produces `trek-domane-road-bike-dublin-XXXXXXXX` format
- Stop word removal: `"A Beautiful Bike For The Road"` should not contain "a", "for", "the"
- Diacritic handling: `"Sean's Raleigh from Baile Atha Cliath"` -- fadas stripped, apostrophe stripped
- Emoji title: `"🚲🚲🚲"` with category `"Bikes"` falls back to `bikes-XXXXXXXX`
- Empty title fallback with no category: produces `listing-XXXXXXXX`
- Truncation: title longer than 60 chars truncates on word boundary
- County omitted when null: slug has no county segment, just `{title}-{shortid}`
- Short ID format: last 8 chars of every generated slug match `/^[a-z0-9]{8}$/`
- `parseSlugShortId`: valid slug returns 8-char string; too-short slug returns null; invalid chars return null
- `isUuidParam`: valid UUID returns true; slug returns false; short string returns false

**Step 1: Create `scripts/backfill-slugs.ts`**

This is a one-off Node.js script that populates slug and short_id for all existing ads that do not have them yet. It runs locally using the Supabase JS client with the service role key.

Implementation:
- Import `createClient` from `@supabase/supabase-js`, and `generateAdSlug` from `../src/lib/server/slugs.ts` (relative import).
- Read `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from environment variables (use `process.env`).
- Create a Supabase client with `auth: { persistSession: false, autoRefreshToken: false }`.
- Fetch all ads where `slug IS NULL` in batches of 100: `select('id, title, category, location_profile_data').is('slug', null).limit(100)`.
- For each ad in the batch:
  - Extract county name from `location_profile_data`: `ad.location_profile_data?.county?.name ?? null`
  - Call `generateAdSlug(ad.title, countyName, ad.category)`
  - Extract short_id: `slug.slice(-8)`
  - Update the row: `.update({ slug, short_id }).eq('id', ad.id)`
  - Handle unique constraint violation (error code `23505`): retry with a new slug (max 3 attempts).
- After all rows are backfilled, run: `ALTER TABLE public.ads ALTER COLUMN slug SET NOT NULL; ALTER TABLE public.ads ALTER COLUMN short_id SET NOT NULL;` using the Supabase client's `.rpc()` or raw SQL. If raw SQL is not available via the JS client, output the ALTER statements for manual execution and print instructions.
- Log progress: `console.log('backfill_progress', { processed, total, batch })`.
- Add a 100ms delay between batches: `await new Promise(r => setTimeout(r, 100))`.

Add a shebang line and make it executable: `#!/usr/bin/env -S npx tsx`. Add instructions as a comment at the top:

3. Implement insert-with-retry for collision handling. Replace the single `.insert()` call with a retry loop (max 3 attempts):
   ```typescript
   const MAX_SLUG_RETRIES = 3;
   let inserted: { id: string; slug: string } | null = null;
   let insErr: any = null;

   for (let attempt = 0; attempt < MAX_SLUG_RETRIES; attempt++) {
     const slug = generateAdSlug(title, countyName, category);
     const shortId = slug.slice(-8);
     const result = await locals.supabase
       .from('ads')
       .insert({
         // ... all existing fields ...
         slug,
         short_id: shortId
       })
       .select('id, slug')
       .single();

     if (!result.error) {
       inserted = result.data;
       insErr = null;
       break;
     }

     // If unique constraint violation on short_id or slug, retry
     if (result.error.code === '23505') {
       log('warn', 'ads_post_slug_collision', { attempt, slug });
       insErr = result.error;
       continue;
     }

     // Other error -- don't retry
     insErr = result.error;
     break;
   }
   ```
