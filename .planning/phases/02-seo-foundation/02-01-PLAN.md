<objective>
Create SEO utility modules and add per-page meta tags, canonical URLs, Open Graph/Twitter Card tags, and JSON-LD Product schema to all existing public pages (ad pages, category pages, homepage). Also create OG fallback placeholder images for social sharing previews when ads have no uploaded images.

Purpose: These are the foundational SEO building blocks. Without unique titles, descriptions, canonicals, OG tags, and structured data, Google cannot properly index pages and social sharing previews are broken. The fallback images ensure social shares never show broken image placeholders.

Output: Three reusable SEO utility modules in `src/lib/seo/`, updated server load functions + svelte:head blocks on ad pages, category pages, and homepage, and 9 static OG fallback images.
</objective>

  <name>Task 1: Create SEO utility modules (meta, jsonld, og)</name>
  <files>src/lib/seo/meta.ts, src/lib/seo/jsonld.ts, src/lib/seo/og.ts</files>
  <action>
Create `src/lib/seo/` directory with three modules:

**meta.ts** -- Title, description, and canonical builders:
- `buildAdTitle(adTitle: string, countyName: string | null): string` -- returns `{adTitle} for Sale in {countyName} | Fogr.ai`. If no county, omit " in {county}".
- `buildCategoryTitle(categoryDisplayName: string, countyName?: string | null): string` -- returns `Second-Hand {Category} for Sale in {countyName} | Fogr.ai` (with county) or `Second-Hand {Category} for Sale in Ireland | Fogr.ai` (without county).
- `buildCountyTitle(countyName: string): string` -- returns `Second-Hand Classifieds in {countyName} | Fogr.ai`.
- `buildHomepageTitle(): string` -- returns `Buy & Sell Second-Hand in Ireland | Fogr.ai \u2014 Fógra\u00ed`.
- `buildDescription(text: string, maxLen?: number): string` -- truncates to ~155 chars at word boundary, appends "..." if truncated. Strip newlines.
- `buildCanonical(origin: string, pathname: string): string` -- returns origin + pathname (no trailing slash, no query params).

**jsonld.ts** -- JSON-LD structured data builders:
- `productJsonLd(ad: AdSeoData, origin: string): Record<string, unknown>` -- builds Product schema per research doc. Include `@context`, `@type: Product`, name, description, url, category, image (if exists), offers with priceCurrency (EUR), availability (InStock or SoldOut for expired), itemCondition (UsedCondition). Only include price in offers if ad.price is a positive number.
- Define `AdSeoData` interface: `{ title, description, slug, price: number | null, currency: string, imageUrl: string | null, category: string, countyName: string | null, isExpired: boolean }`.

**og.ts** -- Open Graph and Twitter Card data builders:
- `buildAdOg(ad: { title: string, description: string, slug: string, imageUrl: string | null, categorySlug: string }, origin: string): OgData` -- og:title = ad title + " for Sale in {county}" (without "| Fogr.ai"), og:description = first ~155 chars, og:image = first image URL or `${origin}/og-fallback/${categorySlug}.png` (absolute URL to static fallback), og:url = canonical URL, og:type = "product".
- `buildCategoryOg(category: string, countyName: string | null, origin: string, pathname: string): OgData` -- similar pattern for category pages.
- `buildHomepageOg(origin: string): OgData` -- homepage Open Graph data.
- Define `OgData` type: `{ title, description, image, url, type, siteName: 'Fogr.ai' }`.

Per user decision: use "Second-Hand" (not "Used"), "Fogr.ai" pipe-separated at end of titles, "Fogr.ai \u2014 Fógra\u00ed" on homepage.

Follow project conventions: named exports, camelCase functions, PascalCase types, single quotes, tabs, no trailing commas. Place in `src/lib/seo/` (new directory).
  </action>
  <verify>Run `npx svelte-check --tsconfig ./tsconfig.json` -- no type errors in the new files. Verify files exist with `ls src/lib/seo/`.</verify>
  <done>Three SEO utility modules exist with typed exports. All functions follow the title/description patterns from user decisions. No type errors.</done>

  ```
  }
  ```

- Add `<svelte:head>` block at the top of the template (before the existing `{#if data?.ad}` block):

  ```svelte
    {#if data?.seo}
      <title>{data.seo.title}</title>
      <meta name="description" content={data.seo.description} />
      <link rel="canonical" href={data.seo.canonical} />
      {#if data.seo.robots}
        <meta name="robots" content={data.seo.robots} />
      {/if}
      <meta property="og:title" content={data.seo.og.title} />
      <meta property="og:description" content={data.seo.og.description} />
      <meta property="og:image" content={data.seo.og.image} />
      <meta property="og:url" content={data.seo.og.url} />
      <meta property="og:type" content={data.seo.og.type} />
      <meta property="og:site_name" content={data.seo.og.siteName} />
      <meta name="twitter:card" content="summary_large_image" />
      <meta name="twitter:title" content={data.seo.og.title} />
      <meta name="twitter:description" content={data.seo.og.description} />
      <meta name="twitter:image" content={data.seo.og.image} />
      {@html `<script type="application/ld+json">${JSON.stringify(data.seo.jsonLd).replace(/</g, '\\u003c')}</script>`}
  ```
