<objective>
Create dynamic sitemap.xml and robots.txt endpoints that serve crawlers with the correct index of public content and access directives.

Purpose: Without a sitemap, Google discovers pages only through crawling links -- slow and incomplete for a user-generated-content site. Without a correctly configured robots.txt, crawlers waste budget on admin pages or index content that should be private.

Output: Two SvelteKit server endpoints replacing/supplementing static files. The existing `static/robots.txt` is either deleted or overridden by the dynamic endpoint.
</objective>

**Important: Research flagged that `super-sitemap`'s filesystem-based route discovery may not work on Cloudflare Workers (no filesystem access). Try super-sitemap first; if it fails at build/runtime, fall back to a hand-rolled sitemap endpoint.**

**Approach A -- Try super-sitemap first:**
- Install: `npm i -D super-sitemap`
- Create `src/routes/sitemap.xml/+server.ts`
- Import `* as sitemap from 'super-sitemap'`
- Create a Supabase client using `platform.env` bindings (same pattern as cron-worker.ts): `PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from `platform.env`
- Query active ads: `SELECT slug, updated_at FROM ads WHERE status = 'active' AND expires_at > now() AND slug IS NOT NULL`
- Build paramValues for super-sitemap:
  - `/(public)/ad/[slug]`: array of ad slugs
  - `/(public)/[category=category]`: array of valid category slugs (from `slugToCategory` reverse map)
  - `/(public)/[county=county]`: array of valid county slugs
  - `/(public)/[category=category]/[county=county]`: array of valid `{category}/{county}` combos
- For programmatic pages: only include category, county, or combo slugs where the listing count >= 3. Query: aggregate count of active ads grouped by category and county_id, then filter combinations with count >= 3.
- excludeRoutePatterns: `['.*\\(app\\).*', '.*/api/.*', '.*/auth/.*', '.*/login.*']`
- Set origin to `https://fogr.ai`
- Return `sitemap.response({...})`

**Approach B -- Fallback hand-rolled sitemap (use if super-sitemap fails on Cloudflare):**
- Build XML manually with proper `<?xml?>` declaration and `<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">`
- Include: homepage, all active ad URLs, category pages with >= 3 listings, county pages with >= 3 listings, category+county combos with >= 3 listings
- Include `<lastmod>` from `updated_at` for ads
- Return as `new Response(xml, { headers: { 'Content-Type': 'application/xml', 'Cache-Control': 'public, max-age=3600' } })`
- Use proper XML escaping for any user-generated content in URLs (slugs should be safe but be defensive)

**County slug resolution for sitemap:**
- Import or recreate the county slug lookup from Plan 02 (the `countySlugToOption` helper or equivalent). The sitemap needs to enumerate all valid county slugs and map them to county IDs for the listing count query.

**Listing count query strategy:**
- Run a single aggregate query: `SELECT category, location_profile_data->>'county'->>'id' as county_id, count(*) FROM ads WHERE status = 'active' AND expires_at > now() GROUP BY category, county_id`
- Build a map of `{category}` -> count, `{county}` -> count, `{category+county}` -> count
- Filter to only include those with count >= 3

**Cache:** Set `Cache-Control: public, max-age=3600` (1 hour). Sitemaps do not need to be real-time.

**Step 1: Create dynamic robots.txt endpoint**

Create `src/routes/robots.txt/+server.ts`:
```typescript
import type { RequestHandler } from '@sveltejs/kit';

export const GET: RequestHandler = async ({ url }) => {
  const body = `User-agent: *
Allow: /

# Block authenticated/private routes
Disallow: /account
Disallow: /admin
Disallow: /ads
Disallow: /messages
Disallow: /post
Disallow: /auth
Disallow: /api
Disallow: /login

# AI Crawlers — explicitly ALLOW to maximise AI discovery
# Being in AI-generated answers and recommendations is a key market entry strategy
User-agent: GPTBot
Allow: /

User-agent: ChatGPT-User
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: anthropic-ai
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Applebot-Extended
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: Bytespider
Allow: /

# Sitemap
Sitemap: ${url.origin}/sitemap.xml
`;

  return new Response(body, {
    headers: {
      'Content-Type': 'text/plain',
      'Cache-Control': 'public, max-age=86400'
    }
  });
};
```

Block the following routes (all under `(app)` group or auth/api):
- `/account` -- user account settings
- `/admin` -- admin panel
- `/ads` -- user's ad management
- `/messages` -- user messaging
- `/post` -- ad posting form
- `/auth` -- OAuth callbacks and logout
- `/api` -- REST API endpoints
- `/login` -- login page (no value to crawlers)

**AI Crawler Strategy:** Explicitly allow all major AI crawlers with dedicated `User-agent` + `Allow: /` blocks. This ensures Fogr.ai listings appear in AI-generated answers (ChatGPT, Claude, Perplexity, Google AI Overviews, Apple Intelligence, etc.). AI bots respect the same private route Disallow rules from the `*` block above. Crawlers to allow: GPTBot, ChatGPT-User, ClaudeBot, anthropic-ai, PerplexityBot, Applebot-Extended, Google-Extended, Bytespider.

**Step 2: Remove static/robots.txt**

Delete `static/robots.txt` to avoid conflict. Per research Pitfall 7: static files take priority over SvelteKit routes in Cloudflare adapter. The static file MUST be removed or the dynamic endpoint will never be reached.

Run: `rm static/robots.txt` (or `git rm static/robots.txt`)
